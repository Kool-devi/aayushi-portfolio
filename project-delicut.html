TLDR

Overview

Delicut is a UAE-based meal plan subscription app serving 15,000+ customers across daily delivery tiers. When I joined, the app had a solid product — but its interface was built for one kind of user: the one who signs up and never deviates. In reality, users skipped deliveries, joined plans late, cancelled mid-cycle, and returned months after their subscription lapsed. None of those moments had a designed screen. I mapped every state a real user could land in and designed the flows that made the app usable for all of them — not just the ideal ones.

The Problem

The app was built for the ideal journey. Real users take a different one.

How do you design a subscription experience that holds up when users pause mid-cycle, skip deliveries, rejoin after months away, or sign up on day three of a seven-day plan?

Before the plan begins

User type New user
Plan status Plan not started
Viewing date Upcoming plan
Available action View plan details

The plan is purchased but deliveries haven't started yet. Users need a clear preview of what's coming — menu, start date, and how to tweak before day one.

Today's delivery is locked

User type New user

Viewing date Today
Available action Changes locked

Once the kitchen cut-off passes, edits are frozen. The screen explains why changes aren't available and points users to what they can still do.
